{traklet:section:objective}
## Objective
The endpoint `/api/report-bug` must implement rate limiting (e.g., 5 requests per IP per minute) and request size limits to prevent abuse as an open relay to GitHub Issues.
{/traklet:section:objective}

{traklet:section:prerequisites}
## Prerequisites
- Endpoint exists and is unauthenticated (by design for public bug reports)
- GITHUB_TOKEN configured (or route falls back to local storage)
{/traklet:section:prerequisites}

{traklet:section:steps}
## Steps
**Rate limit test:**
1. Send 6 POST requests to `/api/report-bug` within 60 seconds from same IP
2. Verify:
   - Requests 1-5 return 200 (accepted)
   - Request 6 returns 429 (Too Many Requests)
   - Request 6 error message: "Rate limit exceeded" or similar

**Size limit test:**
1. POST with 2MB description (very large payload)
2. Verify returns 413 (Payload Too Large) or 400 (size validation error)
3. Verify no GitHub issue created for oversized payload

**XSS prevention:**
1. POST with description containing `<script>alert('xss')</script>`
2. Verify script tags stripped or escaped before storage/GitHub
{/traklet:section:steps}

{traklet:section:expected}
## Expected
- Rate limit: 5 requests per IP per 60 seconds (configurable)
- Size limit: 256 KB max per request (or reasonable limit)
- XSS prevention: HTML stripped or sanitized server-side
- Images array limited to reasonable count (e.g., 10 images max)
- Per-image size limit (e.g., 5MB per image)
- Rate limit uses in-memory LRU cache (or Redis in production)
- Rate limit returns 429 with Retry-After header
- Oversized requests rejected before GitHub API call
- Optional: require captcha for unauthenticated submissions
{/traklet:section:expected}

{traklet:section:diagnostics}
## Diagnostics
Endpoint: app/api/report-bug/route.ts
Current: No rate limiting, no size cap, no input sanitization
Fix: Add rate-limit middleware or in-handler IP-based LRU cache
Fix: Validate request body size before parsing
Fix: Strip HTML with DOMPurify or similar library
Library: Consider @upstash/ratelimit or simple Map-based implementation
{/traklet:section:diagnostics}
