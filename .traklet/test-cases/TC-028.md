{traklet:section:objective}
## Objective
When an admin applies filters to the registrations list (date range, entry point, check-in status), the export should respect those filters and only export matching records.
{/traklet:section:objective}

{traklet:section:prerequisites}
## Prerequisites
- User signed in as org admin
- Organization has registrations with varying dates, entry points, and check-in statuses
- Filter functionality exists (TC-011)
{/traklet:section:prerequisites}

{traklet:section:steps}
## Steps
1. Navigate to `/dashboard/registrations`
2. Apply a filter (e.g., select "Last 7 days" date range)
3. Verify filtered results display in table
4. Note the count of filtered results (e.g., "5 of 20 registrations")
5. Click "Export CSV" button
6. Download and open CSV file
7. Verify CSV contains only the 5 filtered records, not all 20
8. Return to registrations page
9. Apply different filter (e.g., select specific entry point "Main Entrance")
10. Export CSV again
11. Verify new CSV contains only registrations from "Main Entrance"
12. Test export PDF with filters applied
13. Verify PDF also respects filters
{/traklet:section:steps}

{traklet:section:expected}
## Expected
- Export respects all active filters at time of export
- CSV/PDF contains only visible (filtered) records
- Export count matches filtered count shown in UI
- No filtered-out records included in export
- Filter state does not reset after export
- Can export with multiple filters combined (date + entry point + status)
- Export filename includes filter indication (optional)
{/traklet:section:expected}
