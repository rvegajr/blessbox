{traklet:section:objective}
## Objective
Verify that the Traklet GitHub PAT remains server-only and never appears in client JavaScript bundles, maintaining the fix from qa-report/fix-traklet.md.
{/traklet:section:objective}

{traklet:section:prerequisites}
## Prerequisites
- Traklet dev widget enabled (NEXT_PUBLIC_TRAKLET_ENABLED=true in .env.local)
- TRAKLET_PAT set (no NEXT_PUBLIC_ prefix)
- Fresh production build
{/traklet:section:prerequisites}

{traklet:section:steps}
## Steps
1. Run `npm run build` to create production bundle
2. Search client bundle for PAT:
   - `grep -Ra "gho_" .next/static/` (GitHub PAT prefix)
   - `grep -Ra "TRAKLET_PAT" .next/static/`
   - `grep -Ra "<actual PAT value>" .next/static/`
3. Verify 0 matches in .next/static/
4. Verify TRAKLET_PAT only in server chunks:
   - `grep -Ral "TRAKLET_PAT" .next/` should only match server files (.js.map)
5. Open browser DevTools → Sources
6. Search all loaded JS files for "TRAKLET_PAT" or PAT value
7. Verify 0 matches in browser sources
8. Verify /api/dev/traklet-proxy route exists and reads TRAKLET_PAT server-side
9. Verify TrakletDevWidget uses baseUrl pointing to proxy
10. Verify widget passes dummy token: 'proxy' (not real PAT)
{/traklet:section:steps}

{traklet:section:expected}
## Expected
- No PAT literal in .next/static/ (client bundles)
- No NEXT_PUBLIC_TRAKLET_PAT environment variable exists
- TRAKLET_PAT only referenced in server code (app/api/dev/traklet-proxy/route.ts)
- TrakletDevWidget.tsx passes token: 'proxy' to Traklet.init()
- Proxy route at /api/dev/traklet-proxy strips client Authorization header
- Proxy substitutes server-side TRAKLET_PAT before forwarding to api.github.com
- Proxy returns 404 in production (NODE_ENV === 'production')
- app/layout.tsx hard-gates widget render with NODE_ENV !== 'production'
- Production build tree-shakes TrakletDevWidget to null
{/traklet:section:expected}

{traklet:section:evidence}
## Evidence
Fix verified in qa-report/fix-traklet.md:
"Zero client leakage. The 4 hits are .js.map files under .next/server/chunks/ — the source text of the proxy route itself, never shipped to the browser."
{/traklet:section:evidence}
