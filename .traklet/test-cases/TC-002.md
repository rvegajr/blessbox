{traklet:section:objective}
## Objective
A returning user can sign in by entering their email at `/login`, receiving a 6-digit code, and entering it to establish a session.
{/traklet:section:objective}

{traklet:section:prerequisites}
## Prerequisites
- User email already exists in users table
- User has at least one organization membership
- Valid SendGrid configuration
- No active bb_session cookie
{/traklet:section:prerequisites}

{traklet:section:steps}
## Steps
1. Navigate to `/login`
2. Enter email address
3. Click "Send 6-digit code" / "Email me a sign-in link"
4. Wait for email (check inbox and spam folder)
5. Retrieve 6-digit code from email
6. Enter code in verification input
7. Submit code
8. Verify redirect to dashboard or originally requested page
9. Refresh page and confirm session persists
{/traklet:section:steps}

{traklet:section:expected}
## Expected
- Email arrives within 30 seconds
- Valid code → bb_session cookie set with 30-day expiration
- JWT payload contains: sub (user id), email, organizationId (if single org), role
- Protected routes accessible after sign-in
- If user visited protected route before login (e.g., `/dashboard`), redirect back after auth via `?next=` param
- Invalid code → "Invalid verification code" error, no session
- Code expires after 10 minutes
- Session persists across page refreshes until expiration
{/traklet:section:expected}
