{traklet:section:objective}
## Objective
An admin can mark a registration as "checked in" by clicking a check-in button, verify the status changes immediately, and see a timestamp recorded.
{/traklet:section:objective}

{traklet:section:prerequisites}
## Prerequisites
- User signed in as org admin
- At least one registration exists with status "Not Checked In"
{/traklet:section:prerequisites}

{traklet:section:steps}
## Steps
1. Navigate to `/dashboard/registrations` or registration detail page
2. Find a registration showing "Not Checked In" status
3. Click "Check In" button
4. Verify status changes immediately to "Checked In"
5. Verify check-in timestamp appears (date + time)
6. Verify button changes to "Undo Check-in" or disappears
7. Refresh page and verify status persists
8. Check database to confirm:
   - checked_in_at timestamp set
   - checked_in_by user_id recorded (if tracked)
{/traklet:section:steps}

{traklet:section:expected}
## Expected
- Check-in updates database immediately
- Status change reflected in UI without page refresh
- Timestamp format: "Checked in at Jan 10, 2026 2:30 PM" or similar
- Cannot check in same registration twice (button disabled or hidden)
- Check-in action is logged in activity feed
- Statistic cards on dashboard increment check-in count
- QR code scan count separate from check-in count
- Works from both list view and detail view
{/traklet:section:expected}
