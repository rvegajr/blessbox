{traklet:section:objective}
## Objective
The `/api/onboarding/verify-code` endpoint must handle database foreign key constraint errors gracefully without leaking raw SQLite error messages or returning 500 status codes.
{/traklet:section:objective}

{traklet:section:prerequisites}
## Prerequisites
- Organization created but not yet verified
- User attempts to verify code during onboarding
- Database in state that triggers SQLITE_CONSTRAINT_FOREIGNKEY (e.g., missing parent record)
{/traklet:section:prerequisites}

{traklet:section:steps}
## Steps
1. Create organization via `/onboarding/organization-setup`
2. Receive 6-digit verification code
3. Before verifying, simulate foreign key constraint scenario (e.g., delete related record)
4. POST to `/api/onboarding/verify-code` with valid code
5. Observe response status and body
{/traklet:section:steps}

{traklet:section:expected}
## Expected
- Should return 400 or 500 with user-friendly error message
- Should NOT leak raw database error: "SQLITE_CONSTRAINT_FOREIGNKEY"
- Should NOT expose internal database schema details
- Should log full error server-side for debugging
- Primary onboarding path should not trigger this under normal conditions
- If triggered, should guide user to retry or contact support
{/traklet:section:expected}

{traklet:section:evidence}
## Evidence
From qa-report/SUMMARY.md Blocker 9:
"`/api/onboarding/verify-code` 500s with raw `SQLITE_CONSTRAINT_FOREIGNKEY` — primary onboarding path broken + DB internals leaked."
{/traklet:section:evidence}

{traklet:section:diagnostics}
## Diagnostics
Check logs for foreign key constraint violations.
Verify database relationships: organizations → users → memberships.
Confirm verify-code logic wraps DB operations in try/catch with sanitized errors.
{/traklet:section:diagnostics}
