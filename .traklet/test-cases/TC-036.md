{traklet:section:objective}
## Objective
The endpoint `POST /api/system/clear-database` must require super-admin authentication, explicit confirmation parameter, and environment flag, preventing accidental or malicious database wipes.
{/traklet:section:objective}

{traklet:section:prerequisites}
## Prerequisites
- Endpoint exists at /api/system/clear-database
- Database has production data
{/traklet:section:prerequisites}

{traklet:section:steps}
## Steps
**Anonymous attack scenario:**
1. No authentication
2. POST `/api/system/clear-database`
3. Observe response

**Expected fix verification:**
1. Returns 401 if not authenticated
2. Returns 403 if not super-admin
3. Returns 400 if no confirmation parameter (e.g., `{confirm: true}` or `{confirmText: "DELETE ALL DATA"}`)
4. Returns 403 if NODE_ENV === 'production' and no special ALLOW_PROD_WIPE flag
5. Only proceeds if ALL conditions met: super-admin + confirmation + environment check
{/traklet:section:steps}

{traklet:section:expected}
## Expected
- Requires authentication (bb_session with role: superadmin)
- Requires explicit confirmation in request body
- Requires environment flag or NODE_ENV !== 'production' check
- Returns 401/403 if any check fails
- Logs clear-database attempts (who, when, IP)
- Preserves super-admin account as specified in route
- Shows scary confirmation dialog in UI before calling endpoint
- Ideally disabled completely in production (404) unless explicitly enabled
{/traklet:section:expected}

{traklet:section:evidence}
## Evidence
