{traklet:section:objective}
## Objective
A user can complete full payment flow using Square sandbox test card, verify payment processes successfully, subscription activates, and payment is recorded in database.
{/traklet:section:objective}

{traklet:section:prerequisites}
## Prerequisites
- User signed in
- Square sandbox configured (SQUARE_ENVIRONMENT=sandbox)
- Square Payment Form loaded (iframe from squarecdn.com)
{/traklet:section:prerequisites}

{traklet:section:steps}
## Steps
1. Navigate to `/checkout?plan=standard` (no coupon)
2. Verify total shows $19.00
3. Verify Square Payment Form iframe loads
4. Fill in Square sandbox test card:
   - Card Number: 4111 1111 1111 1111 (Visa success card)
   - CVV: 123
   - Expiry: 12/25 (or any future date)
   - Postal Code: 12345
5. Click "Complete Payment" button
6. Wait for payment processing (loading state)
7. Verify redirect to dashboard within 15 seconds
8. Verify success message or toast notification
9. Check subscription status on dashboard: "Standard - Active"
10. Verify in database:
    - subscriptions table has new record
    - payment_id populated (Square payment ID)
    - plan_type = 'standard'
    - status = 'active'
{/traklet:section:steps}

{traklet:section:expected}
## Expected
- Square Payment Form loads without CSP errors
- Test card 4111... processes successfully in sandbox
- Payment ID returned from Square API
- Subscription created immediately after successful payment
- User redirected to dashboard on success
- Payment record includes Square transaction ID
- Billing cycle starts from payment date
- User gains access to Standard plan features immediately
- Sandbox payment not charged to real card
- Can test decline scenarios with specific test cards (e.g., 4000 0000 0000 0002)
{/traklet:section:expected}
