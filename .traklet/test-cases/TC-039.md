{traklet:section:objective}
## Objective
Verify that Content-Security-Policy nonce value changes on every page request, preventing nonce reuse attacks.
{/traklet:section:objective}

{traklet:section:prerequisites}
## Prerequisites
- CSP middleware active (middleware.ts)
- Server running
{/traklet:section:prerequisites}

{traklet:section:steps}
## Steps
1. Load page A (e.g., /dashboard) and inspect:
   - Response header: Content-Security-Policy
   - Extract nonce value from script-src (e.g., 'nonce-abc123...')
   - Save nonce1
2. Refresh page A (hard refresh, Cmd+Shift+R)
3. Extract new nonce value from CSP header
4. Save nonce2
5. Compare nonce1 vs nonce2
6. Verify nonce1 !== nonce2 (different values)
7. Load different page B (e.g., /pricing)
8. Extract nonce3
9. Verify nonce3 !== nonce1 && nonce3 !== nonce2
10. Verify nonce format: randomUUID() with hyphens removed (32 hex chars)
{/traklet:section:steps}

{traklet:section:expected}
## Expected
- Each request generates unique nonce via crypto.randomUUID()
- Nonce appears in CSP header: `script-src ... 'nonce-{nonce}' 'strict-dynamic'`
- Nonce exposed via x-nonce request header (for server components)
- No 'unsafe-inline' in CSP script-src (replaced by nonce)
- Next.js Flight payload scripts include matching nonce attribute
- Inline scripts without nonce are blocked
- Nonce collision probability negligible (UUID v4 entropy)
- Middleware runs on all routes except _next/static and assets
{/traklet:section:expected}

## Diagnostics
Middleware: middleware.ts
CSP generation: Per-request via crypto.randomUUID()
Nonce format: `nonce.replace(/-/g, '')` (UUID without hyphens)
Test: curl -v http://localhost:7777/ | grep -i content-security-policy
Verify: Each curl shows different nonce value
