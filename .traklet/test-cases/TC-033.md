{traklet:section:objective}
## Objective
The `/api/payment/process` endpoint must validate payment amount server-side against the selected plan's actual price, rejecting client-supplied amounts that don't match.
{/traklet:section:objective}

{traklet:section:prerequisites}
## Prerequisites
- User signed in
- Standard plan costs $19.00
{/traklet:section:prerequisites}

{traklet:section:steps}
## Steps
**Attack scenario:**
1. Navigate to `/checkout?plan=standard` (shows $19.00)
2. Intercept POST to `/api/payment/process` using browser DevTools
3. Modify request body: `{planType:'standard', amount:1}` (change $19 to $0.01)
4. Send modified request
5. Observe response and database

**Expected fix verification:**
1. POST with modified amount returns 400 or 403
2. Error message: "Invalid amount for selected plan" or similar
3. No payment processed
4. No subscription created
5. Database unchanged
{/traklet:section:steps}

{traklet:section:expected}
## Expected
- Server MUST recalculate amount from planType, ignoring client `amount`
- If client `amount` doesn't match server calculation, reject with 400
- Cannot charge 1¢ for Standard plan by tampering request
- Payment processing uses server-calculated amount only
- Square API receives server-validated amount, not client amount
- Subscription record shows correct base_amount from plan config
{/traklet:section:expected}

{traklet:section:evidence}
## Evidence
From qa-report/SUMMARY.md Blocker 1:
"`/api/payment/process` trusts client `amount` — POST `{planType:'enterprise', amount:1}` charges 1¢."
{/traklet:section:evidence}

{traklet:section:diagnostics}
## Diagnostics
Endpoint: app/api/payment/process/route.ts
Fix: Retrieve plan amount from database/config, ignore client amount
Validation: Zod schema should not accept `amount` from client, or validate it matches server value
{/traklet:section:diagnostics}
