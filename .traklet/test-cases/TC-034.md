{traklet:section:objective}
## Objective
The endpoints `/api/registrations/[id]` (GET/PUT/DELETE) must require authentication and verify the registration belongs to the user's organization, preventing cross-org PII access.
{/traklet:section:objective}

{traklet:section:prerequisites}
## Prerequisites
- Two organizations exist: Org A (user's org) and Org B (other org)
- Org B has registration with known UUID
- User signed in to Org A
{/traklet:section:prerequisites}

{traklet:section:steps}
## Steps
**IDOR attack scenario:**
1. User signed in to Org A
2. Obtain registration UUID from Org B (e.g., via guessing, enumeration, or leaked)
3. GET `/api/registrations/{orgBRegistrationUuid}`
4. Observe response

**Expected fix verification:**
1. GET returns 401 if not authenticated
2. GET returns 403 or 404 if registration not in user's org
3. PUT returns 403 or 404 (cannot modify other org's registration)
4. DELETE returns 403 or 404 (cannot delete other org's registration)
5. Unauthenticated request returns 401 (not 200 with data)
{/traklet:section:steps}

{traklet:section:expected}
## Expected
- All registration [id] endpoints require authentication
- All requests validate `registration.organization_id === user.organization_id`
- Cross-org access returns 403 Forbidden or 404 Not Found (404 preferred to not leak existence)
- Unauthenticated access returns 401 Unauthorized
- No PII leaked in error messages
- Super-admin can access all registrations (special case)
{/traklet:section:expected}

{traklet:section:evidence}
## Evidence
From qa-report/SUMMARY.md Blocker 4:
"`/api/registrations/[id]` GET/PUT/DELETE — no auth, cross-org PII read/update/delete by UUID."
{/traklet:section:evidence}

{traklet:section:diagnostics}
## Diagnostics
Endpoints: app/api/registrations/[id]/route.ts
Fix: Add middleware auth check or getServerSession() at top of handlers
Fix: Join registrations with organizations, filter by current user's org
Validation: Test with two different org accounts
{/traklet:section:diagnostics}
