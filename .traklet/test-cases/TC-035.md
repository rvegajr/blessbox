{traklet:section:objective}
## Objective
The endpoint `GET /api/registrations/export?orgId=...` must require authentication and verify the user belongs to the requested organization, preventing unauthenticated bulk PII export.
{/traklet:section:objective}

{traklet:section:prerequisites}
## Prerequisites
- Organization exists with registrations containing PII
- User NOT signed in (anonymous)
{/traklet:section:prerequisites}

{traklet:section:steps}
## Steps
**Anonymous attack scenario:**
1. No authentication (no bb_session cookie)
2. GET `/api/registrations/export?orgId={knownOrgId}`
3. Observe response

**Expected fix verification:**
1. Request returns 401 Unauthorized if not authenticated
2. If authenticated but orgId != user's org, returns 403 Forbidden
3. No CSV data returned without proper authorization
4. Super-admin can export any org (special case)
{/traklet:section:steps}

{traklet:section:expected}
## Expected
- Endpoint requires authentication (checks session)
- Endpoint validates user is member of requested organization
- Anonymous access returns 401, no data
- Cross-org access returns 403, no data
- Only org admins can export their own org's data
- Super-admin can export any org (if that's intended behavior)
- No PII leaked in error responses
{/traklet:section:expected}

{traklet:section:evidence}
## Evidence

{/traklet:section:evidence}

{traklet:section:diagnostics}
## Diagnostics
Endpoint: app/api/registrations/export/route.ts or similar
Fix: Add getServerSession() check at handler start
Fix: Validate user.organizationId === query.orgId
Test: curl without cookies should return 401
{/traklet:section:diagnostics}
