{traklet:section:objective}
## Objective
During onboarding, an organization admin can build a custom registration form by adding fields (text, email, phone, textarea, date), marking them required/optional, reordering them, and saving the form configuration.
{/traklet:section:objective}

{traklet:section:prerequisites}
## Prerequisites
- User signed in (via TC-001 or TC-002)
- User on form builder step of onboarding wizard
{/traklet:section:prerequisites}

{traklet:section:steps}
## Steps
1. Verify form builder page loaded (`data-testid="form-builder"`)
2. Add fields using "Add Field" button:
   - First Name (Text, Required)
   - Last Name (Text, Required)
   - Email (Email, Required)
   - Phone (Phone, Optional)
   - Address (Textarea, Optional)
   - Date of Birth (Date, Optional)
3. Test field reordering (drag-and-drop or up/down buttons)
4. Preview form to verify field order and labels
5. Click "Save Form" or "Continue"
6. Verify form saved to database (form_configs table)
7. Verify redirect to next onboarding step (QR configuration)
{/traklet:section:steps}

{traklet:section:expected}
## Expected
- All field types (text, email, phone, textarea, date) available
- Required/optional toggle works for each field
- Field reordering updates preview in real-time
- Form preview matches configured field order
- Save persists form_config JSON to database
- Form accessible later at `/register/{orgSlug}/{qrSlug}` with same field order
- Cannot proceed without at least one required field
{/traklet:section:expected}
