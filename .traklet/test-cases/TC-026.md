{traklet:section:objective}
## Objective
An admin can export registrations as a CSV file, download it, open it in a spreadsheet program, and verify all registration data is accurately included with proper headers and UTF-8 encoding.
{/traklet:section:objective}

{traklet:section:prerequisites}
## Prerequisites
- User signed in as org admin
- Organization has multiple registrations
{/traklet:section:prerequisites}

{traklet:section:steps}
## Steps
1. Navigate to `/dashboard/registrations`
2. Locate "Export" button or "Export CSV" button (id="export-csv")
3. Click the export CSV button
4. Verify file download starts
5. Verify filename format: `registrations-{orgSlug}-{timestamp}.csv` or similar
6. Open downloaded CSV file in Excel/Google Sheets
7. Verify CSV contains:
   - Header row with column names (Registration ID, First Name, Last Name, Email, Phone, Entry Point, Check-in Status, Registration Date, etc.)
   - All registration records as rows
   - Data matches what's shown in dashboard
8. Verify special characters render correctly (UTF-8 encoding with BOM)
9. Verify dates formatted consistently (ISO 8601 or human-readable)
10. Verify no CSV injection (cells starting with =, +, -, @ are properly escaped)
{/traklet:section:steps}

{traklet:section:expected}
## Expected
- CSV downloads successfully
- File opens correctly in spreadsheet programs
- UTF-8 BOM included for Excel compatibility
- All columns present: ID, names, email, phone, custom fields, timestamps, check-in status
- Custom form fields included as additional columns
- No data truncation or corruption
- Formula injection prevention: cells starting with =, +, -, @ prefixed with single quote
- Empty fields represented as empty cells (not "null" or "undefined")
- Boolean values: "Yes"/"No" or "true"/"false"
- Commas in data properly escaped with quotes
{/traklet:section:expected}
