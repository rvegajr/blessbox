{traklet:section:objective}
## Objective
The `/api/payment/process` endpoint must NOT use `body.email` to provision subscriptions, preventing an attacker from purchasing plans for arbitrary email addresses without authentication.
{/traklet:section:objective}

{traklet:section:prerequisites}
## Prerequisites
- Endpoint accepts payment processing
- User may or may not be authenticated
{/traklet:section:prerequisites}

{traklet:section:steps}
## Steps
**Auth-bypass attack scenario:**
1. User NOT authenticated (or authenticated as different user)
2. POST `/api/payment/process` with body:
   ```json
   {
     "planType": "enterprise",
     "amount": 99,
     "email": "victim@example.com",
     "paymentMethodId": "valid_square_token"
   }
   ```
3. Observe response and database

**Expected fix verification:**
1. If not authenticated, returns 401 Unauthorized
2. If authenticated, uses session.user.email (NOT body.email)
3. body.email parameter ignored or causes 400 error
4. Subscription created only for authenticated user's email
5. Cannot provision subscription for arbitrary email by passing it in body
{/traklet:section:steps}

{traklet:section:expected}
## Expected
- Endpoint requires authentication (session must exist)
- Uses `session.user.email` from authenticated session
- Ignores or rejects `body.email` if it doesn't match session email
- Cannot purchase plan for someone else's email
- Subscription and organization provisioning use session identity only
- Combined with TC-033: also validates amount server-side
- Result: Cannot get free Enterprise for any email via body.email bypass
{/traklet:section:expected}

{traklet:section:evidence}
## Evidence
From qa-report/SUMMARY.md Blocker 2:
"`/api/payment/process` auth-bypass via `body.email` — provisions org for any email; combined with #1 = free Enterprise."
{/traklet:section:evidence}

{traklet:section:diagnostics}
## Diagnostics
Endpoint: app/api/payment/process/route.ts
Current: May trust body.email for org/subscription creation
Fix: Require getServerSession() at handler start
Fix: Use session.user.email for all provisioning logic
Fix: Remove or validate body.email matches session.user.email
Validation: POST with mismatched email should return 403
{/traklet:section:diagnostics}
