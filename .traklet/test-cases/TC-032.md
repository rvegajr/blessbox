{traklet:section:objective}
## Objective
An admin can enroll a participant in a class, verify they appear in the roster, attempt to enroll another participant when capacity is reached, and see a "capacity reached" error message.
{/traklet:section:objective}

{traklet:section:prerequisites}
## Prerequisites
- User signed in as org admin
- Class created with capacity = 1 (for testing)
- Two participants exist (Alice and Bob from TC-031)
{/traklet:section:prerequisites}

{traklet:section:steps}
## Steps
1. Navigate to `/classes/{classId}` (class detail page)
2. Locate "Enroll Participant" section with dropdown
3. Select first participant (Alice) from dropdown (select option by index 1)
4. Click "Enroll" button
5. Verify enrollment success message or roster update
6. Verify "Roster" heading appears
7. Verify Alice appears in roster list
8. Attempt to enroll second participant (Bob):
   - Select Bob from dropdown (index 2)
   - Click "Enroll" button
9. Verify error message: "Capacity reached" or "Class is full"
10. Verify Bob NOT added to roster (enrollment rejected)
11. Check database: enrollments table shows 1 enrollment (Alice only)
{/traklet:section:steps}

{traklet:section:expected}
## Expected
- First enrollment succeeds (capacity = 1, enrolled = 1)
- Enrolled participant appears in roster immediately
- Second enrollment attempt rejected (capacity = 1, enrolled = 1, attempting = 2)
- Error message clear: "Capacity reached" or similar
- Capacity check happens before database insert
- No race condition: concurrent enrollments don't bypass capacity
- Roster shows accurate count: "1 / 1 enrolled"
- Cannot enroll same participant twice in same class
- Capacity enforcement uses transaction or WHERE clause to prevent races
{/traklet:section:expected}
