{traklet:section:objective}
## Objective
An admin can search registrations by name or email, filter by date range, entry point, or check-in status, and clear filters to see all registrations again.
{/traklet:section:objective}

{traklet:section:prerequisites}
## Prerequisites
- User signed in as org admin
- Organization has multiple registrations with varying attributes
{/traklet:section:prerequisites}

{traklet:section:steps}
## Steps
1. Navigate to `/dashboard/registrations`
2. Locate search box (placeholder: "Search by name or email...")
3. Type a name (e.g., "John")
4. Verify results filter in real-time to show only matching registrations
5. Clear search and verify all registrations reappear
6. Test email search (type partial email)
7. If date filter exists:
   - Select date range (e.g., last 7 days)
   - Verify only registrations in that range appear
8. If entry point filter exists:
   - Select specific entry point
   - Verify only registrations from that QR code appear
9. If status filter exists:
   - Select "Checked In"
   - Verify only checked-in registrations appear
   - Select "Not Checked In"
   - Verify only unchecked registrations appear
10. Click "Clear Filters" or reset button
11. Verify all registrations reappear
{/traklet:section:steps}

{traklet:section:expected}
## Expected
- Search works on first name, last name, and email
- Search is case-insensitive
- Search updates results immediately (or with short debounce)
- Filters can be combined (e.g., date range + entry point)
- Filter state persists during session (survives page refresh)
- Clear filters button resets all filters at once
- Result count updates to show "X of Y registrations"
- No results state shown if filters match nothing
{/traklet:section:expected}
