{traklet:section:objective}
## Objective
The checkout page correctly rejects invalid, expired, and max-usage coupons with clear error messages, and allows user to try again.
{/traklet:section:objective}

{traklet:section:prerequisites}
## Prerequisites
- User signed in on checkout page
- Test coupons exist in database:
  - EXPIRED (expired coupon)
  - MAXEDOUT (max_uses reached)
  - INVALID (non-existent code)
{/traklet:section:prerequisites}

{traklet:section:steps}
## Steps
**Test Case A: Expired Coupon**
1. Navigate to `/checkout?plan=standard`
2. Enter coupon code: "EXPIRED"
3. Click "Apply"
4. Verify error message: "Coupon expired" or "Invalid coupon"
5. Verify no discount applied
6. Verify can try different coupon

**Test Case B: Max Uses Reached**
1. Clear previous coupon (if any)
2. Enter coupon code: "MAXEDOUT"
3. Click "Apply"
4. Verify error: "Coupon limit reached" or "Maximum uses"
5. Verify no discount applied

**Test Case C: Invalid Code**
1. Clear previous coupon
2. Enter coupon code: "INVALID"
3. Click "Apply"
4. Verify error: "Invalid coupon code"
5. Verify no discount applied

**Test Case D: Empty Code**
1. Clear coupon field
2. Click "Apply" with empty field
3. Verify validation error or no action
{/traklet:section:steps}

{traklet:section:expected}
## Expected
- All invalid coupon attempts rejected
- Error messages are clear and user-friendly
- No discount applied for invalid coupons
- User can try multiple coupons in same session
- No payment processed with invalid coupon
- Error messages match type: "expired" / "limit reached" / "invalid code"
- Coupon field remains accessible after error
{/traklet:section:expected}
